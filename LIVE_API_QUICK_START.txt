HOW IT WORKS
================================================================================

1. User requests cost data for a city
2. Backend checks 1-hour cache
   - If found → return cached data (< 10ms)
   - If not found → continue
3. Fetch Numbeo public page for city
4. Parse HTML to extract 50+ price items
5. Calculate monthly estimates for 6 categories
6. Cache result for 1 hour
7. Return live data with source="numbeo_live"

If Numbeo is unavailable:
   → Use hardcoded fallback data
   → Return with source="fallback"
   → Still works! ✅

================================================================================
TROUBLESHOOTING
================================================================================

Q: Getting fallback data instead of live?
A: Numbeo might be rate-limiting. Wait a few seconds and retry.
   Cache will serve subsequent requests.

Q: Endpoint returns error?
A: Check if city name is correct (case-sensitive).
   Supported cities: Bangalore, Mumbai, Delhi, etc.

Q: Response is slow?
A: First request fetches from Numbeo (200-500ms).
   Subsequent requests use cache (< 10ms).

Q: Want to clear cache?
A: Cache auto-clears after 1 hour.
   Or restart backend to clear immediately.
