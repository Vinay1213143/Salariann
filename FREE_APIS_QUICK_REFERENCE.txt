API DETAILS
================================================================================

NUMBEO API
  Endpoint: https://www.numbeo.com/api/city_prices
  Method: GET
  Parameters:
    - api_key: Your API key
    - query: "City, Country" (e.g., "Bangalore, India")
  
  Example:
  curl "https://www.numbeo.com/api/city_prices?api_key=YOUR_KEY&query=Bangalore,India"

TELEPORT API
  Endpoint: https://api.teleport.org/v1/cities/
  Method: GET
  Parameters:
    - search: City name
  
  Example:
  curl "https://api.teleport.org/v1/cities/?search=Bangalore"

ADZUNA API
  Endpoint: https://api.adzuna.com/v1/api/jobs/in/search
  Method: GET
  Parameters:
    - app_id: Your app ID
    - app_key: Your app key
    - location: City name
    - results_per_page: Number of results
    - page: Page number
  
  Example:
  curl "https://api.adzuna.com/v1/api/jobs/in/search?app_id=YOUR_ID&app_key=YOUR_KEY&location=Bangalore"

================================================================================
TROUBLESHOOTING
================================================================================

Problem: "Unable to fetch cost of living data"
Solution:
  1. Check NUMBEO_API_KEY in .env
  2. Verify API key is correct
  3. Test API directly: curl "https://www.numbeo.com/api/city_prices?api_key=YOUR_KEY&query=Bangalore,India"
  4. Check Numbeo API status
  5. System will use fallback (Teleport or hardcoded)

Problem: Getting "fallback" instead of "live"
Solution:
  1. API keys not configured (expected)
  2. Add API keys to .env
  3. Restart backend
  4. Verify with: curl "http://localhost:3001/api/cost-of-living/Bangalore" | jq '.source'

Problem: Slow responses
Solution:
  1. First request is slower (API call)
  2. Subsequent requests are fast (cached)
  3. Cache TTL is 1 hour
  4. Check API status if consistently slow

Problem: API key not working
Solution:
  1. Verify key is correct (copy-paste)
  2. Check API website for key
  3. Regenerate key if needed
  4. Test API directly
  5. Check API documentation
